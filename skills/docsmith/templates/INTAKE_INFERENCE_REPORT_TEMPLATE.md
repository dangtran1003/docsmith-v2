# Intake Inference Report Template

Generated by `/docsmith init --from-source` and `/docsmith module --from-source`. Provides transparency about how AI filled the intake from external source documents (BA docs, PRDs, existing docs).

> Introduced in v1.5.9. The intake form remains the single source of truth — this report is an audit log so BAs can review what AI inferred, what it guessed, and what it defaulted.

## File location

```
documentation/intake/.inference/<timestamp>-<scope>.md
```

Where `<scope>` is `project` (for `init --from-source`) or `<module-name>` (for `module --from-source`). Files retained for audit; new run produces new file.

## How to use this report

1. **Open `documentation/intake/project.md`** — that's the file you'll edit.
2. **In project.md, look for `← AI guess` or `← needs verify` markers** — these correspond to "Guesses" rows above. Verify and adjust if AI got it wrong.
3. **Defaults are usually fine for first run.** Only override if you have a specific reason (regulated industry → formal voice, marketing-grade → AI voiceover).
4. **Don't edit this report file.** Re-running `init --from-source` regenerates it with new timestamp.
5. **Re-run `init --from-source` when BA doc updates.** Existing project.md merged via re-run protocol — your manual edits preserved.

## Re-run behavior

When `init --from-source` runs and `project.md` already exists:

1. Fetch sources fresh
2. Re-infer fields
3. Compare with current intake values
4. Show diff: what AI would change vs what BA edited manually
5. Re-run protocol gate:
   - **Update** (default): only fields BA didn't manually edit get re-inferred values
   - **Overwrite**: discard BA edits, use new inference (rare; for major source rewrites)
   - **Skip**: keep existing intake; report shows what would have changed
6. Generate new inference report with updated timestamp

User edits are preserved by hash-tracking: AI computes hash of each field's value at last inference; if current hash matches, field was untouched and can be re-inferred; if hash differs, BA edited and field is preserved.

## Limitations

- **AI confidence is heuristic** — "Facts" can still be wrong if BA doc has typos or outdated info. Verify critical fields.
- **No deep code analysis** — when source is GitHub repo, AI reads README and surface code structure; doesn't analyze every function. For deep code-driven docs, consider manual scope refinement.
- **Defaults are conservative** — AI defaults to safest choice (silent video, standalone deploy) even when BA doc hints at other strategies. Override explicitly.
- **Module detection is heuristic** — AI groups features into modules based on source structure (sections, headings, file paths). May produce unexpected groupings; review module list before running pipeline.
